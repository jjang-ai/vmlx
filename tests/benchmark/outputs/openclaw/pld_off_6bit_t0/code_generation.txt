### 2. **Reversal Algorithm: Iterative Approach**
I chose the **iterative approach** over recursive for several reasons:
- **Space efficiency**: O(1) space vs O(n) stack space for recursion
- **No stack overflow risk**: Safe for very long lists
- **Clearer control flow**: Easier to understand and debug

**Algorithm Steps:**
1. Initialize `prev = None`, `current = head`
2. For each node:
   - Save `current.next` in a temporary variable
   - Reverse the pointer: `current.next = prev`
   - Advance both pointers: `prev = current`, `current = temp`
3. Return `prev` (the new head)
