# Pre-PR Checklist for Big Features

Here's what I'd recommend checking before opening your PR:

## Code Quality
- ✅ All tests pass (existing + new)
- ✅ Code builds without errors or warnings
- ✅ Follows project's style guidelines
- ✅ No debug code, TODOs, or commented-out blocks left behind

## Feature Completeness
- ✅ Feature is fully functional end-to-end
- ✅ Edge cases are handled
- ✅ Error states are properly managed
- ✅ Works with existing features (no regressions)

## Documentation
- ✅ Code is well-commented where needed
- ✅ Public APIs have docstrings/descriptions
- ✅ Updated relevant docs (README, API docs, etc.)
- ✅ Clear PR description explaining what and why

## Technical Concerns
- ✅ Performance considered (especially for data-heavy operations)
- ✅ Security reviewed (input validation, auth, etc.)
- ✅ Breaking changes identified and documented
- ✅ Migration steps provided if needed

## PR Hygiene
- ✅ Commits are logical and clean (consider squashing)
- ✅ Branch is up-to-date with main
- ✅ Appropriate reviewers identified
- ✅ Linked to relevant issues/tickets

## Bonus: Self-Review
- ✅ Read through your own code fresh
- ✅ Test the feature manually one more time
- ✅ Ask yourself: "Would I want to review this?"

---

**Quick tip:** For big features, consider opening a "work in progress" PR first to get early feedback, then refine before the final review.
