Temperature is a hyperparameter that controls the randomness of token selection during generation.

At temperature = 0:
- The model becomes completely deterministic
- It always selects the token with the highest probability (argmax)
- Same input always produces the same output
- No randomness or exploration
- Can be good for tasks requiring exact reproducibility
- May produce more "safe" but potentially less creative outputs
- Can get stuck in repetitive patterns

At temperature = 0.3:
- Low but non-zero temperature
- Introduces some randomness while still favoring high-probability tokens
- The probability distribution is slightly "flattened" compared to temperature 0
- More diverse outputs across multiple runs

- May discover better solutions through exploration
- Still relatively controlled - not as random as higher temperatures

The tradeoff:
- Temperature 0: Maximum determinism, reproducibility, but potentially less creative/diverse
- Temperature 0.3: Slight randomness for diversity while maintaining quality, but outputs vary between runs

Use cases:
- Temperature 0: Code generation, mathematical reasoning, when exact reproducibility matters

# Temperature Tradeoff: 0 vs 0.3

## What Temperature Does

Temperature controls randomness in token selection. It modifies the probability distribution before sampling:

```
Adjusted probability ∝ Original probability^(1/temperature)
```

## Temperature = 0 (Greedy Decoding)

**Behavior:**
- Always picks the highest-probability token
- Completely deterministic — same input = same output every time

**Pros:**
- Maximum reproducibility
- Most "confident" predictions
- Good for factual/analytical tasks

**Cons:**
- Can produce repetitive or generic text
- May miss better answers that require exploration
- Can get stuck in loops

## Temperature = 0.3 (Low Randomness)

**Behavior:**
- Mostly picks high-probability tokens
- Occasionally selects lower-probability options
- Outputs vary slightly between runs

**Pros:**
- More natural, varied text
- Avoids repetitive patterns
- Can discover better responses through exploration

**Cons:**
- Non-deterministic (different output each time)
- Slightly less predictable quality

## Practical Tradeoff

| Use Case | Recommended |
|----------|-------------|
| Code generation, math | 0 |
| Factual Q&A | 0–0.2 |
| General text generation | 0.3–0.7 |
| Creative writing | 0.7–1.0+ |

**Rule of thumb:** Use 0 when you need exact reproducibility; use 0.3 when you want coherent but slightly varied outputs.
