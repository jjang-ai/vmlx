This is a comprehensive checklist to ensure your Pull Request is high-quality, easy to review, and ready for merge.

### 1. Code Quality & Functionality
*   **[ ] Does it work?** Run the application locally and manually test the new feature.
*   **[ ] Are there edge cases?** Test what happens if inputs are empty, invalid, or network requests fail.
*   **[ ] Did you delete any temporary `console.log` statements or debug code?**
*   **[ ] Is the code clean?** Check for unused variables, imports, or commented-out blocks of code.
*   **[ ] Are variable and function names descriptive?**
*   **[ ] Is the code DRY (Don't Repeat Yourself)?**

### 2. Testing
*   **[ ] If you added a new feature, did you add corresponding Unit Tests?**
*   **[ ] If you changed existing logic, did you update the existing Tests?**
*   **[ ] Do all tests pass locally?** (Run `npm test`, `pytest`, etc.)

### 3. The "Big Feature" Specifics
*   **[ ] Did you rebase or merge the main branch?** Ensure you haven't created merge conflicts by working on an old version of the code.
*   **[ ] Is the feature behind a Feature Flag?** If the feature isn't 100% complete but you need to merge the code, wrap it in a flag so it doesn't break production.
*   **[ ] Did you check for performance impacts?** Since it's a big feature, did you introduce heavy loops or large database queries?

### 4. Documentation
*   **[ ] Did you update the README?** If the feature changes how the app is configured or used.
*   **[ ] Did you update API documentation?** (e.g., Swagger/OpenAPI) if endpoints changed.
*   **[ ] Are complex logic blocks commented?** Explain *why* you did something, not just *what*.

*   **[ ] Did you check the Diff?** Look at the files changed. Did you accidentally modify a file you didn't mean to?
