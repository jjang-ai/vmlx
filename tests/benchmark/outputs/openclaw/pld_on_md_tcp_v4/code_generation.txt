Let me think about the implementation:

For reversing a linked list, I can use an iterative approach which is more efficient than recursive. The algorithm:
- Keep track of three pointers: previous, current, and next
- Iterate through the list, reversing each node's next pointer
- Return the new head of the reversed list

Let me write clean, well-documented code with a good example.

```python
class Node:
    """A node in a singly linked list."""
    def __init__(self, data):
        self.data = data
        self.next = None

    def __repr__(self):
        return f"Node({self.data})"

def reverse_linked_list(head):
    """
    Reverses a singly linked list iteratively.

    Args:
        head (Node): The head node of the linked list.

    Returns:
        Node: The new head node of the reversed linked list.
    """
    previous = None
    current = head

    while current:
        next_node = current.next  # Store the next node
        current.next = previous   # Reverse the link
        previous = current        # Move previous forward
        current = next_node       # Move current forward

    return previous

def print_linked_list(head):
    """
    Prints the linked list in a readable format.

    Args:
        head (Node): The head node of the linked list.
    """
    values = []
    current = head
    while current:
        values.append(str(current.data))
        current = current.next
    print(" -> ".join(values))

def create_linked_list(values):
    """
    Creates a linked list from a list of values.

    Args:
        values (list): List of values to create nodes from.

    Returns:
        Node: The head node of the created linked list.
    """
    if not values:
        return None
