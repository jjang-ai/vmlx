    *   **Topic:** Temperature parameter in Large Language Model (LLM) inference.
    *   **Specific Values:** Compare Temperature = 0 vs. Temperature = 0.3.
    *   **Context:** LLM inference server (implies production, API usage, consistency vs. creativity).
    *   **Goal:** Explain the tradeoff (pros/cons, use cases).

2.  **Define "Temperature" (Briefly):**
    *   What does it actually do? It controls the randomness of token selection.
    *   Formula intuition: $P(token) \propto \exp(\logits / T)$.
    *   $T=0$: Deterministic (argmax).
    *   $T>0$: Probabilistic (softmax sampling).

3.  **Analyze Temperature = 0 (Deterministic):**
    *   *Mechanism:* Always picks the token with the highest probability.
    *   *Pros:* Reproducibility, consistency, precision, factuality (usually), lower latency (sometimes, due to caching strategies).
    *   *Cons:* Repetitive, rigid, prone to "mode collapse" (getting stuck in loops), less creative, might fail at open-ended tasks.
    *   *Best for:* Coding, math, factual Q&A, structured output (JSON), debugging.

4.  **Analyze Temperature = 0.3 (Low Randomness):**
    *   *Mechanism:* Softmax sampling with a slight bias towards high-probability tokens, but allows lower-probability alternatives.
    *   *Pros:* More natural flow, avoids repetition better than 0, slight variation for similar prompts, better for creative writing (though still constrained).
    *   *Cons:* Less reproducible (same prompt $\neq$ same output), slightly higher hallucination risk than 0, slightly more compute (sampling overhead).
    *   *Best for:* Chatbots, summaries, general conversation, creative tasks where variety is needed but accuracy matters.
