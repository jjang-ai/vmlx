2.  **Define the Core Concepts:**
    *   **KV Cache (Key-Value Cache):**
        *   *What is it?* A mechanism to store the attention keys and values computed during the generation of a sequence.
        *   *Why use it?* To avoid recomputing attention for previous tokens during autoregressive generation. It's the standard optimization for *single* sequence generation.
        *   *Scope:* Specific to the current request/session.
        *   *Lifecycle:* Created per request, discarded when the request finishes.
    *   **Prefix Cache (Prompt/Context Cache):**
        *   *What is it?* A mechanism to store the KV cache of a *common prefix* shared across multiple requests.
        *   *Why use it?* To avoid recomputing attention for the prompt (input) when multiple requests share the same starting text (e.g., system prompts, user templates, RAG context).
        *   *Scope:* Shared across *multiple* requests/clients.
        *   *Lifecycle:* Persisted in a cache store, retrieved when a matching prefix is detected.

3.  **Identify Key Differences (The "Vs." Structure):**
    *   *Purpose:* Single request optimization vs. Multi-request optimization.
    *   *Granularity:* Per-token generation vs. Per-prompt block.
    *   *Sharing:* None vs. Shared across requests.
    *   *Storage:* Volatile (per session) vs. Persistent (shared store).
    *   *Use Case:* Standard inference vs. Batched inference with repeated prompts (e.g., chatbots with system prompts, code completion).
