2.  **Define "Temperature" in the Context of LLMs:**
    *   What is it? A hyperparameter controlling randomness/entropy in token selection.
    *   How does it work? It scales the logits before the softmax function.
    *   Low temp (0): Deterministic, greedy decoding (always picks the highest probability token).
    *   High temp (>1): Stochastic, creative, diverse (flattens probability distribution).
    *   Mid temp (0.3): Slight randomness, maintains coherence but allows variation.

3.  **Analyze Temperature = 0:**
    *   *Behavior:* Greedy decoding.
    *   *Pros:* Reproducible (same input = same output), deterministic, high precision for factual tasks, good for code/logic, low latency (sometimes, though sampling overhead is negligible compared to generation).
    *   *Cons:* Repetitive, rigid, prone to getting stuck in loops (rare but possible), lacks nuance, might hallucinate confidently if the model is unsure but picks the "most likely" wrong token.
    *   *Use Cases:* Coding, math, summarization (where accuracy is key), API responses requiring consistency.

4.  **Analyze Temperature = 0.3:**
    *   *Behavior:* Low randomness sampling (e.g., Top-K/Top-P usually accompany this, but focusing on temp alone).
    *   *Pros:* More natural language flow, less robotic, handles ambiguity better, reduces repetition, introduces slight variation for roleplay or creative tasks without going off the rails.
    *   *Cons:* Non-deterministic (same input might yield slightly different outputs), slightly higher risk of hallucination compared to 0 (though negligible at 0.3), harder to cache responses exactly.
    *   *Use Cases:* Chatbots, creative writing, customer support (where tone matters), open-ended Q&A.
