2.  **Analyze the Code:**
    ```python
    def find_dupes(lst):
        dupes = []
        for i in range(len(lst)):
            for j in range(i+1, len(lst)):
                if lst[i] == lst[j] and lst[i] not in dupes:
                    dupes.append(lst[i])
        return dupes
    ```
    *   **Functionality:** It iterates through all pairs of elements. If two elements match and the element isn't already in the `dupes` list, it adds it.
    *   **Logic:** It finds duplicate values.
    *   **Edge Cases:** Empty list, single element, all unique, all same.
    *   **Complexity:** Nested loops -> O(n^2). The `in` check on `dupes` adds another factor (up to O(n)), making the inner check O(n). Total complexity roughly O(n^3) in worst case if `dupes` grows large, though practically often less. Wait, `lst[i] not in dupes` is O(k) where k is length of `dupes`. So inner loop is O(n) for comparison + O(k) for check. Outer loop O(n). Total O(n^2 * k). Since k <= n, it's O(n^3) worst case. Actually, the comparison `lst[i] == lst[j]` is O(1) for primitives.
