Problem: Write a Python function that reverses a linked list. Include the Node class definition and a short example in __main__.

Approach: to reverse a singly linked list is the **iterative approach**. It is efficient, easy to understand, and avoids the potential stack overflow issues associated with recursive solutions on very large lists.

Key considerations:
- The `Node` class needs a `value` and a `pointer` to the next node.
- The `reverse_linked_list` function needs to take the `head` of the list and return the new `head`.
- We need three pointers to perform the swap:
       `previous`: Initially `None`, will eventually become the new head.
       `current`: Initially the `head`, traverses the list.
       `next_node`: A temporary storage for the next node in the list so we don't lose the reference when we reverse the link.
- Loop through the list:
       Store the `next_node`.
       Point `current.next` to `previous` (the reversal step).
       Move `previous` to `current`.
       Move `current` to `next_node`.
- Return `previous` as the new head.

Complexity: - Time: O(n) - We visit each node exactly once.
- Space: O(1) - We only use a constant amount of extra space for the pointers.
