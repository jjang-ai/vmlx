# Pre-PR Checklist for Large Features

## 🧪 Testing & Quality

- [ ] **All tests pass** - Run the full test suite locally
- [ ] **Added tests for new code** - Unit tests, integration tests, E2E tests as appropriate
- [ ] **Tested edge cases** - Empty states, error states, boundary conditions
- [ ] **Manual testing complete** - Walked through the entire feature flow
- [ ] **Regression testing** - Verified existing features still work

## 📝 Code Review Self-Check

- [ ] **Code follows team conventions** - Naming, formatting, architecture
- [ ] **No debug code left** - Console.logs, debugger statements, temporary code
- [ ] **Functions are appropriately sized** - No god functions/classes
- [ ] **Error handling is comprehensive** - Graceful failures with helpful messages
- [ ] **No hardcoded values** - Configuration, environment variables where appropriate
- [ ] **Type safety** - Proper types defined (if using TypeScript or similar)

## 📚 Documentation

- [ ] **README updated** - If feature affects setup or usage
- [ ] **API documentation** - If new endpoints or interfaces
- [ ] **Inline comments** - For complex logic that needs explanation
- [ ] **Migration guide** - If breaking changes or data migrations needed
- [ ] **Changelog entry** - Summary of what changed

## 🔍 Feature-Specific Checks

- [ ] **Performance considered** - No N+1 queries, reasonable payload sizes
- [ ] **Security reviewed** - Input validation, authentication/authorization
- [ ] **Accessibility checked** - ARIA labels, keyboard navigation, color contrast
- [ ] **Mobile responsive** - If UI-related
- [ ] **Database migrations tested** - Rollback scenarios
