Let me analyze this function systematically.

**Function Analysis:**
```python
def find_dupes(lst):
    dupes = []
    for i in range(len(lst)):
        for j in range(i+1, len(lst)):
            if lst[i] == lst[j] and lst[i] not in dupes:
                dupes.append(lst[i])
    return dupes
```

**CORRECTNESS:**
- Does it find duplicates? Yes
- Test case 1: `[1, 2, 3]` → `[]` ✓
- Test case 2: `[1, 1, 2]` → `[1]` ✓
- Test case 3: `[1, 2, 1, 2]` → `[1, 2]` ✓
- Test case 4: `[1, 1, 1]` → `[1]` ✓
- Test case 5: `[]` → `[]` ✓
- Test case 6: `[1]` → `[]` ✓
- Edge case: Unhashable types (lists) - the `in` check would work but comparison might be slow
- The function correctly identifies each duplicate value only once

**PERFORMANCE:**
- Time complexity: O(n²) for nested loops + O(n) for each `not in dupes` check
- Worst case: O(n² × m) where m is number of duplicates
- Can be improved to O(n) using a set
- The `lst[i] not in dupes` check inside the inner loop is particularly inefficient

**STYLE:**
- Function name `find_dupes` is informal; `find_duplicates` is more professional
- Variable name `dupes` same issue
- Using `range(len())` is not
