# Reversing a Linked List in Python

## System Design Considerations

### 1. **Algorithm Selection**
- **Iterative approach** chosen over recursive for:
  - O(1) space complexity (recursive would be O(n) due to call stack)
  - No risk of stack overflow on large lists
  - Better performance in production environments

### 2. **Edge Cases Handled**
- Empty list (None head)
- Single node list
- Two node list
- Lists with cycles (would need separate detection)

### 3. **Thread Safety**
- For multi-threaded environments, consider adding locks around the reversal operation
- Current implementation is not thread-safe (race conditions possible)

### 4. **Memory Management**
- In-place reversal: No new nodes allocated
- Python's garbage collector
