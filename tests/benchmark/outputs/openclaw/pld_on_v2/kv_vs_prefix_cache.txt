# KV Cache vs. Prefix Cache: A Detailed Comparison

## Overview

Both **KV cache** and **prefix cache** cache intermediate computation results during transformer inference to avoid redundant calculations, but they operate at different levels of granularity and serve different purposes.

---

## KV Cache (Key-Value Cache)

### What it is:
A KV cache stores the **key (K) and value (V) matrices** computed during the attention mechanism for each token in a sequence.

### How it works:
```
For each new token generated:
1. Compute Q, K, V for the new token
2. Use cached K, V from previous tokens for attention
3. Cache new K and V for future use
4. Attention = softmax(Q · K^T / √d) · V
```

### Visual representation:
``````
Token 1: K₁, V₁ ─┐
Token 2: K₂, V₂ ─┤
  Token 3: K₃, V₃ ─┼──► KV Cache
  Token 4: K₄, V₄ ─┘
