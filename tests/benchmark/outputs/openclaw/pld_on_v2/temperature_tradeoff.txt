# Temperature Tradeoff: 0 vs 0.3 for LLM Inference

## Understanding Temperature

Temperature is a parameter that controls the randomness of token selection during LLM generation. The formula modifies token probabilities as:

$$P_{adjusted}(token) = \frac{P_{original}(token)^{1/T}}{\sum P_{original}(token)^{1/T}}$$

Where:
- **T = 1**: Original distribution
- **0 < T < 1**: Sharpens distribution → more deterministic
- **T = 0**: Pure greedy selection

---

## Temperature = 0

**Behavior:**
- Always selects the token with the highest probability (greedy decoding)
- Completely deterministic — same input always produces identical output
- No exploration of alternative token choices

**Advantages:**
- ✅ Maximum reproducibility
- ✅ Best for factual/analytical tasks requiring precision
- ✅ Predictable latency (no stochastic variation)
- ✅ Optimal for code generation where exactness matters

**Disadvantages:**
- ❌ Can produce repetitive, rigid outputs
- ❌ May get stuck in local optima
- ❌ Lacks creativity and natural variation
- ❌ Can sound robotic or formulaic

---

## Temperature = 0.3

**Behavior:**
- Introduces mild randomness while heavily favoring high-probability tokens
- Low-probability tokens have a small but non-zero chance of selection
- Same input can produce different outputs across runs

**Advantages:**
- ✅ More natural, human-like variation
- ✅ Better for open-ended creative tasks
- ✅ Reduces repetition in long-form generation
- ✅ Often produces more engaging conversational responses

**Disadvantages:**
- ❌ Slightly less reproducible
- ❌ May occasionally produce suboptimal token choices
- ❌ Marginally more unpredictable for critical applications

---

## Practical Comparison

| Use Case | Recommended | Reason |

- **Code generation**: 0 | Exactness required |
- **Mathematical reasoning**: 0 | Deterministic correctness |
- **Factual Q&A**: 0 | Accuracy priority |
- **Creative writing**: 0.3 | Needs variation |
- **Chat/conversation**: 0.3 | Natural dialogue |
- **Content generation**: 0.3 | Avoid repetition |
- **Summarization**: 0 | Stay close to source |

---

## Recommendation

**Choose Temperature 0 when:**
- Reproducibility is critical
- Task requires factual precision
- Running automated pipelines/tests

**Choose Temperature 0.3 when:**
- ✅ Human-like naturalness matters
- ✅ Creative or open-ended tasks
- ✅ You want to avoid repetitive outputs
- ✅ Conversational applications
