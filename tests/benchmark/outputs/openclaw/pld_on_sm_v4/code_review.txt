CORRECTNESS:  
The function correctly identifies duplicate values in the list and returns them in a list. However, there is a logical flaw: if a value appears multiple times, it will only be added to `dupes` once, which is correct. The condition `lst[i] not in dupes` ensures that each value is included only once. However, the function does not guarantee that duplicates are found in the order of first occurrence, and it may miss duplicates if the list is large or if the duplicates are not adjacent. More importantly, the function does not handle the case where the same value appears multiple times but is only detected once due to the nested loop structure — this is actually correct behavior for deduplication. The main issue is that `lst[i] not in dupes` checks membership in a list that grows dynamically, which is inefficient and not necessary. Despite this, the logic for detecting duplicates is sound and produces the correct result for the given examples.

PERFORMANCE:  
The function has a time complexity of O(n²) due to the nested loops, and for each comparison, it performs a membership check (`lst[i] not in dupes`) which is O(k) where k is the size of `dupes`. In the worst case, this results in O(n³) time complexity. This is highly inefficient for large lists. A more efficient solution would use a set to track seen values, reducing the lookup time to O(1), resulting in O(n) time complexity. The current implementation is unnecessarily slow and scales poorly.
