

## Code Review: `find_dupes`

---

### CORRECTNESS

The function is **correct**. It properly identifies all duplicate values in a list and returns each duplicate exactly once.

**Verification with examples:**
- `find_dupes([1, 2, 3, 2, 4, 1])` → `[1, 2]` ✓
- `find_dupes([1, 1, 1, 1])` → `[1]` ✓
- `find_dupes([1, 2, 3])` → `[]` ✓
- `find_dupes([])` → `[]` ✓

The logic correctly:
1. Compares each pair of elements
2. Checks if the value is already recorded before adding
3. Returns the list of duplicates

---

### PERFORMANCE

The function has **O(n²) time complexity** due to nested loops, and **O(n) space complexity** for the result.

**Issues:**
- The nested loop approach compares every pair, doing up to n(n-1)/2 comparisons
- The `lst[i] not in dupes` check inside the loop adds another O(k) operation per match (where k = number of duplicates found)

**Better approach:** Use a set for O(n) time:
```python
def find_dupes(lst):
    seen = set()
    dupes = set()
    for item in lst:
        if item in seen:
            dupes.add(item)
        else:
            seen.add(item)
    return list(dupes)
```
