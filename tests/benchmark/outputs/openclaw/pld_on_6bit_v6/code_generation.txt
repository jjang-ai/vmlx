# Reversing a Linked List in Python

## System Design Considerations

### 1. **Algorithm Selection**
- **Iterative Approach**: O(n) time, O(1) space - preferred for production
- **Recursive Approach**: O(n) time, O(n) space due to call stack - elegant but risks stack overflow on large lists

### 2. **Edge Cases to Handle**
- Empty list (None head)
- Single node list
- Two node list
- Lists with cycles (optional validation)

### 3. **Memory Safety**
- In-place reversal avoids allocating new nodes
- No memory leaks in Python (garbage collected), but important in other languages

### 4. **Type Hints & Documentation**
- Improves code maintainability and IDE support
- Clear contracts for function behavior
