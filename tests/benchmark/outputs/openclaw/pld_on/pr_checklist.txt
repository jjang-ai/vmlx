# Pre-PR Checklist for Big Features

Here's what I'd recommend checking:

## Code Quality
- [ ] All tests pass (unit, integration, e2e)
- [ ] New tests added for the feature
- [ ] Linting/formatting clean
- [ ] No new warnings or errors

## Feature Completeness
- [ ] Feature works end-to-end
- [ ] Edge cases handled
- [ ] Error states tested
- [ ] Documentation updated (README, inline comments, API docs)

## Review-Friendly
- [ ] PR description explains *what* and *why*
- [ ] Screenshots/recordings if UI changes
- [ ] Consider splitting into smaller PRs if possible
- [ ] Commits are logical and clean (rebase/squash if needed)

## Technical Debt
- [ ] No `TODO` comments you'll forget
- [ ] No temporary hacks left in
- [ ] Performance considered for scale
- [ ] Security reviewed if applicable

## Process
- [ ] CI/CD pipeline green
- [ ] Database migrations tested (if any)
- [ ] Backwards compatibility checked
- [ ] Got early feedback from a teammate?

---

**One more thing:** For *really* big features, consider sharing a draft PR or getting informal feedback before the official review. It saves everyone time.
