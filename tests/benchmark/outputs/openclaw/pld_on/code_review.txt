**Function Analysis:**
```python
def find_dupes(lst):
    dupes = []
    for i in range(len(lst)):
        for j in range(i+1, len(lst)):
            if lst[i] == lst[j] and lst[i] not in dupes:
                dupes.append(lst[i])
    return dupes
```

**CORRECTNESS:**
- The function finds duplicate elements in a list
- Outer loop iterates through each element
- Inner loop compares each element with subsequent elements
- When a match is found and the element isn't already in dupes, it's added
- Edge cases:
  - Empty list: returns [] ✓
  - No duplicates: returns [] ✓
  - All same elements: returns [element] ✓
  - Multiple different duplicates: returns list of duplicates ✓
- The function is CORRECT

**PERFORMANCE:**
- Time complexity: O(n²) for nested loops + O(n) for "not in" check on list
  - Actually "not in" on list is O(k) where k is number of duplicates found
  - So worst case is O(n² × k) but typically analyzed as O(n²)
- Space complexity: O(k) where k is number of unique duplicates
- Can be improved to O(n) using a set or Counter
- The "not in dupes" check on a list is O(k) - could use a set for O(1)
