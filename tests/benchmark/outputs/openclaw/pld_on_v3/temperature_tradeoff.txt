# Temperature Tradeoff: 0 vs 0.3 in LLM Inference

## What Temperature Does

Temperature scales the probability distribution before token selection:

```
P(token) = softmax(logits / temperature)
```

Lower temperature → sharper distribution → more deterministic
Higher temperature → flatter distribution → more random

---

## Temperature = 0 (Greedy Decoding)

| Aspect | Behavior |
|--------|----------|
| Selection | Always picks the highest-probability token |
| Reproducibility | 100% deterministic — same input = same output |
| Output Quality | Maximizes likelihood of each individual token |
| Risk | Can produce repetitive, "stuck" text |

**Best for:** Code generation, math problems, factual Q&A, production systems requiring exact reproducibility

---

## Temperature = 0.3 (Low Randomness)

| Aspect | Behavior |
|--------|----------|
| Selection | Mostly high-probability tokens, occasional surprises |
| Reproducibility | Different outputs on each run |
| Output Quality | May find better global solutions by exploring |
| Risk | Slightly less predictable |

**Best for:** Conversation, creative writing, open-ended tasks, when variety is valuable
