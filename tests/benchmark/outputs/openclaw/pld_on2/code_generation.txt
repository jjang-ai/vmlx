## Architectural Decisions & Reasoning

### 1. **Algorithm Choice: Iterative Approach**
I chose the iterative approach over recursion because:
- **Space Efficiency**: O(1) space vs O(n) stack space for recursion
- **No Stack Overflow Risk**: Safe for very long lists
- **Better Performance**: No function call overhead

### 2. **Three-Pointer Technique**
The classic approach uses three pointers:
- `prev`: Points to the previous node (initially `None`)
- `current`: Points to the current node being processed (initially `head`)
- `next_temp`: Temporarily stores the next node before we break the link

### 3. **In-Place Reversal**
The algorithm reverses pointers in-place without allocating new nodes, making it memory-efficient.

### 4. **Edge Case Handling**
The algorithm naturally handles:
- Empty list (returns `None`)
- Single node (returns the same node)
- Two or more nodes (full reversal)
