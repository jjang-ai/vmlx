# Pre-PR Checklist for Big Features

Here's a comprehensive checklist to review before opening your PR:

## Code Quality
- [ ] All tests pass (unit, integration, E2E)
- [ ] New tests written for the feature
- [ ] Code follows project style guidelines
- [ ] No debug code, console logs, or TODOs left behind
- [ ] Linting/formatting passes

## Feature Completeness
- [ ] Feature is fully implemented (no partial functionality)
- [ ] Edge cases are handled
- [ ] Error states are properly managed
- [ ] Manual testing completed in realistic scenarios

## Documentation
- [ ] PR description clearly explains:
  - What the feature does
  - Why it was built
  - How to test it
- [ ] Code is adequately commented
- [ ] User-facing documentation updated
- [ ] API/docs generated if applicable

## Integration & Impact
- [ ] No breaking changes (or they're documented)
- [ ] Existing features still work (regression testing)
- [ ] Database migrations tested (if applicable)
- [ ] Configuration changes documented

## Technical Review
- [ ] Performance considerations addressed
- [ ] Security implications reviewed
- [ ] Accessibility checked (if UI)
- [ ] Dependencies documented

## PR Hygiene
- [ ] Commit history is clean and logical
- [ ] Branch is up to date with main
- [ ] Correct reviewers assigned
- [ ] Appropriate labels added
