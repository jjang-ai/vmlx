# Reversing a Linked List in Python

## Architectural Decisions

### 1. **Iterative vs Recursive Approach**
I chose the **iterative approach** because:
- **O(1) space complexity** vs O(n) for recursive (call stack)
- No risk of stack overflow on very long lists
- More intuitive and easier to debug

### 2. **Three-Pointer Technique**
The algorithm uses three pointers:
- `prev`: Points to the previous node (initially `None`)
- `current`: Points to the current node being processed (initially `head`)
- `next_temp`: Temporarily stores the next node before we break the link

### 3. **In-Place Modification**
The list is reversed in-place without allocating new nodes, making it memory-efficient.

### 4. **Edge Case Handling**
The function naturally handles:
- Empty list (`head is None`)
- Single-node list
- Two-node list
