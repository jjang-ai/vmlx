Let me analyze this function thoroughly across three dimensions:

**Function Purpose:**
The function `find_dupes` finds duplicate values in a list and returns them as a list (each duplicate value appears only once in the result).

**CORRECTNESS Analysis:**
- The algorithm uses nested loops to compare each pair of elements
- When a match is found (lst[i] == lst[j]), it checks if already in dupes list
- Only adds to dupes if not already present
- Returns the list of duplicates

Test cases to verify:
- Empty list: returns [] ✓
- No duplicates: returns [] ✓
- Single duplicate pair: [1,2,1] returns [1] ✓
- Multiple duplicates: [1,2,1,2,3] returns [1,2] ✓
- Multiple occurrences of same value: [1,1,1,1] returns [1] ✓
- Works with any hashable type ✓

The function is CORRECT.

**PERFORMANCE Analysis:**
- Time complexity: O(n²) for nested loops + O(n) for each "not in" check on dupes list
- The "lst[i] not in dupes" check is O(k) where k is number of duplicates found so far
- Worst case: O(n² * k) which approaches O(n³) in pathological cases
- Space complexity: O(k) where k is number of unique duplicates

Better approach would use a set:
- Track seen elements with a set O(1) lookup
- Track duplicates with a set O(1) lookup
- Overall O(n) time complexity
