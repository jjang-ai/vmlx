# Pre-PR Checklist for Big Features

## Code Quality
- ✅ **Build passes** - No compilation errors
- ✅ **All tests pass** - Existing + new tests for your feature
- ✅ **Test coverage** - Edge cases, error states, happy paths
- ✅ **Linting/formatting** - Follows project conventions
- ✅ **No debug code** - Removed console.logs, breakpoints, TODOs

## Feature Completeness
- ✅ **Functionality verified** - Manually tested the full user journey
- ✅ **Error handling** - Graceful failures with clear messages
- ✅ **Backwards compatibility** - No unexpected breaking changes
- ✅ **Performance** - No obvious regressions or N+1 queries

## Documentation
- ✅ **Code comments** - Complex logic explained
- ✅ **API docs** - Updated if interfaces changed
- ✅ **README/usage docs** - If user-facing
- ✅ **Migration notes** - If breaking changes exist

## PR Hygiene
- ✅ **Clear title & description** - What, why, how to test
- ✅ **Screenshots/GIFs** - For UI changes
- ✅ **Linked issues** - References to tickets/bugs
- ✅ **Commits squashed** - Logical history (optional but nice)
- ✅ **Rebased** - On latest main/develop

## Before You Hit "Create PR"
- ✅ **Assigned reviewers** - People who know the codebase area
- ✅ **Labels applied** - Feature, breaking-change, etc.
- ✅ **Draft vs Ready** - Consider marking as draft if not 100%

**Pro tip:** For *really* big features, consider splitting into smaller PRs or doing a "design review" PR first with just the architecture/interfaces.
