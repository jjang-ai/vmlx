# Temperature Tradeoff: 0 vs 0.3 for LLM Inference

## Understanding Temperature

Temperature is a sampling parameter that controls the randomness of token selection during LLM generation. It modifies the probability distribution over possible next tokens.

**The formula:** `P(token) = softmax(logits / temperature)`

---

## Temperature = 0 (Greedy/Deterministic)

**Behavior:**
- The model selects the **highest-probability token** at each step
- **Deterministic**: Same input always produces identical output
- Maximizes coherence and factual accuracy
- Minimizes surprises

**Best for:**
- Code generation
- Mathematical reasoning
- Factual Q&A
- Production systems requiring reproducibility
- Search/query tasks

---

## Temperature = 0.3 (Low Randomness)

**Behavior:**
- **Softened probability distribution**
- Introduces **controlled randomness**
- Allows exploration of near-top alternatives
- Adds variety while maintaining quality

**Best for:**
- Creative writing
- Conversational chatbots
- Content generation
- Open-ended questions
- Scenarios valuing variety over exactness

---

## Key Tradeoffs

| Aspect | Temperature 0 | Temperature 0.3 |
|--------|---------------|-----------------|
| **Predictability** | ✅ Perfect reproducibility | ⚠️ Different outputs each run |
| **Creativity** | ❌ Rigid, formulaic | ✅ Moderate variety |
| **Coherence** | ✅ Maximum | ✅ High (slightly reduced) |
| **Factual Accuracy** | ✅ Optimal | ✅ Very good |
