## Architectural Decisions

**Approach**: I'll use the **iterative three-pointer technique** rather than recursion because:
- **O(1) space complexity** vs O(n) for recursive stack frames
- No risk of stack overflow on very long lists
- More efficient and production-friendly

**Design choices**:
1. **In-place reversal**: Modifies the existing list structure rather than creating new nodes
2. **Type hints**: Added for better code documentation and IDE support
3. **Edge case handling**: Explicitly handles empty lists and single-node lists
