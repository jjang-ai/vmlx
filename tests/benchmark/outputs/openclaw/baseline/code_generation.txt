        current.next = prev       Point current node to previous node

## Architectural Decisions & Reasoning

### Data Structure Design
- **Node Class**: A simple singly linked list node with `val` and `next` attributes
- **Choice of Singly Linked List**: Most common interview problem variant; doubly linked lists would require updating two pointers per node

### Algorithm Selection: Iterative Approach
I chose the **iterative 3-pointer technique** over recursion because:
1. **Space Efficiency**: O(1) extra space vs O(n) stack space for recursion
2. **No Stack Overflow Risk**: Safe for very long lists
3. **Performance**: No function call overhead

### Algorithm Steps
```
Initial:  1 → 2 → 3 → None
prev=None, current=1

Step 1:   prev=1 ←, current=2
          1 ← None

Step 2:   prev=2 ← 1, current=3
          2 ← 1 ← None

Step 3:   prev=3 ← 2 ← 1, current=None
          3 ← 2 ← 1 ← None

Result:   3 → 2 → 1 → None (prev is new head)
```
