1. Code quality checks:
- Does the code compile/build successfully?
- Are all tests passing?
- Have they written tests for the new feature?
- Is the code following the project's style guidelines?
- Have they done any refactoring to improve code quality?

2. Feature completeness:
- Is the feature actually complete?
- Are there any TODOs or incomplete parts?
- Have they tested edge cases?
- Is error handling appropriate?

3. Documentation:
- Is there documentation for the new feature?
- Are there inline comments where needed?
- Is there a clear PR description?
- Are there migration notes if needed?

4. Integration:
- Does the feature work with existing code?
- Are there any breaking changes?
- Have they checked for conflicts with other branches?

5. Performance:
- Are there any performance implications?
- Have they done any benchmarking if needed?

6. Security:
- Are there any security implications?
- Have they reviewed for common vulnerabilities?

7. PR preparation:
- Is the commit history clean?
- Is the PR description clear?
- Have they assigned reviewers?
- Have they added appropriate labels?

# Pre-PR Checklist for Big Features

## Code Quality
- [ ] All tests pass (existing + new)
- [ ] Code follows project style guidelines
- [ ] No linting/formatting errors
- [ ] Build succeeds cleanly
- [ ] No console errors/warnings

## Feature Completeness
- [ ] Feature is actually complete (no TODOs left)
- [ ] Edge cases handled
- [ ] Error states tested
- [ ] Works across different environments/browsers

## Testing
- [ ] Unit tests written for new code
- [ ] Integration tests if applicable
- [ ] Manual testing completed
- [ ] Regression testing on related areas

## Documentation
- [ ] Code comments where needed
- [ ] API documentation updated
- [ ] README/usage docs updated
- [ ] Migration notes if breaking changes

## Review Prep
- [ ] Commit history is clean/logical
- [ ] PR description is clear with context
- [ ] Screenshots/videos if UI changes
- [ ] Proper reviewers assigned
- [ ] Labels/categories added

## Integration
- [ ] No breaking changes (or documented)
- [ ] Conflicts resolved with main branch
- [ ] Feature flags in place if needed

## Bonus: Big Feature Specific
- [ ] Performance considered
- [ ] Security reviewed
- [ ] Accessibility checked
- [ ] Database migrations tested (if applicable)

**Pro tip:** For big PRs, consider splitting into smaller, logical PRs if possible. Easier to review, faster feedback, less merge conflict risk.
