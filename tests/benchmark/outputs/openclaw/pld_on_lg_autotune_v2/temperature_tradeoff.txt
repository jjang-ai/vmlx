# Temperature Tradeoff: 0 vs 0.3 for LLM Inference

## Understanding Temperature

Temperature is a sampling parameter that controls the randomness of token selection during LLM generation. The formula for token selection probability is:

$$P(token) = \frac{exp(logit/temperature)}{\sum exp(logit/temperature)}$$

## Temperature = 0 (Deterministic/Greedy)

**Behavior:**
- Always selects the token with the highest probability (argmax)
- Completely deterministic — same input always produces identical output
- No exploration of alternative token choices

**Advantages:**
- ✅ Perfect reproducibility
- ✅ Maximum consistency for factual/technical tasks
- ✅ Predictable outputs for production systems
- ✅ Best for code generation and math

**Disadvantages:**
- ❌ Can produce repetitive or rigid text
- ❌ May miss creative or nuanced alternatives
- ❌ Can feel mechanical in conversational contexts

## Temperature = 0.3 (Low Randomness)

**Behavior:**
- Introduces mild stochasticity while heavily favoring high-probability tokens
- Low-probability tokens have a small but non-zero chance of being selected
- Same input can produce different outputs across runs

**Advantages:**
- ✅ More natural, human-like variation
- ✅ Better fluency in longer generations
- ✅ Avoids repetitive patterns
- ✅ Mild creativity while maintaining coherence

**Disadvantages:**
- ❌ Non-deterministic (different outputs each run)
- ❌ Slightly reduced factual consistency
