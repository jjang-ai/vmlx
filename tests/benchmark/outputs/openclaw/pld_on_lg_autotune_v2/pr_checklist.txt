# Pre-PR Checklist for Big Features

## Code Quality
- [ ] All tests pass (unit, integration, e2e)
- [ ] Linting/formatting passes
- [ ] No new warnings or errors
- [ ] Code coverage is reasonable for new code

## Feature Verification
- [ ] Feature works as intended in staging/preview
- [ ] Edge cases are handled
- [ ] Error states are graceful
- [ ] Performance is acceptable

## Documentation
- [ ] Code is self-documenting or well-commented
- [ ] Public APIs have docstrings
- [ ] README updated if needed
- [ ] Migration guide if breaking changes exist

## PR Hygiene
- [ ] Clear, descriptive title
- [ ] Description explains *what* and *why*
- [ ] Links to related issues/tickets
- [ ] Screenshots for UI changes
- [ ] Breaking changes clearly called out

## Collaboration
- [ ] Team is aware this is coming
- [ ] Appropriate reviewers assigned
- [ ] Consider splitting if too large (>400-500 lines)

## Technical Debt
- [ ] No TODOs left without tracking issues
- [ ] Temporary workarounds identified
- [ ] Security considerations addressed

---

**Quick tip:** For really big features, consider opening a draft PR early to get feedback incrementally rather than waiting until everything is "done."
